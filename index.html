Шпаргалки за командами з Git

  • config --global user.name "Ім'я" - становлює ім'я, яке буде відображатися в поле автора у виконуваних вами коммітов
  • git config --global user.email "адрес ел.пошти" - встановлює адресу електронної пошти, який буде відображатися в інформації про виконувані вами коммітов
  • git init - створює новий локальний репозиторій з заданим ім'ям
  • git status - перераховує всі нові або змінені файли, які потребують фіксації
  • git add - індексує вказаний файл для подальшого коммітів
  • git commit - фіксує проіндексовані зміни і зберігає їх в історію версій
  • git log - історія коммітів для поточної гілки
  • git checkout - перемикається на обрану гілку і оновлює робочу директорію до її стану
  • git branch - список іменованих гілок коммітів з обраною гілкою
  • git push - завантажує всі зміни локальної гілки в віддалений репозиторій
  • git pull - завантажує історію з віддаленого сховища та об'єднує її з локальної
  • git merge - вносить зміни зазначеної гілки в поточну гілку
  • git clone - викачує репозиторій разом з усією його історією змін

Gitignore

Git буде ігнорувати файли і директорії, перераховані у файлі за допомогою wildcard синтаксису

Advanced

  • git revert - створює новий rомміт, який скасовує зміни з попереднього коммітов
  • git reset - ця команда може змінювати існуючу історію (шляхом зміни фіксації, пов'язаної з гілкою)
  • git rebase - ви можете взяти всі зміни, які потрапили в коммітив на одній з гілок, і повторити їх на іншій
  • git cherry-pick - дозволяє забрати один комміт з іншої гілки і вставити його в поточну
  • git stash - тимчасово зберігає всі незафіксовані зміни відслідковуються файлів
